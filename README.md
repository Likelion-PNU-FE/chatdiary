브랜치는 기능따라 말고 작성자에 따라 나눌까 합니다. 소규모일경우 기능따라 나눠도 관리가 잘 안되는 것 같더라구요

## 파일구조
- assets : 이미지나 svg font등의 파일
- components : 컴포넌트들을 모아놓은 폴더
- pages : 페이지들을 모아놓은 폴더
- styles : 전역적으로 사용되는 스타일을 모아놓은 폴더
- api : api 요청을 모아놓은 폴더
- utils : 유틸리티 함수들을 모아놓은 폴더
- hook : 커스텀 훅을 모아놓은 폴더
